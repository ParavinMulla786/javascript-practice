📘 JavaScript Data Types

🔹 Categories

  • Primitive Data Types
    • String
    • Number
    • Boolean
    • Undefined
    • Null
    • BigInt
    • Symbol
  • Non-Primitive Data Types
    • Object
    • Array
    • Function

🔹 Simple Explanation

  • String: Text values → "Hello"
  • Number: Numbers → 10, 10.5
  • Boolean: true / false
  • Undefined: Declared but no value
  • Null: Empty value (intentional)
  • Object: Data in key-value form
  • Array: List of values
  • Function: Reusable code block
  • BigInt: Very large numbers
  • Symbol: Unique identifier

🔹 Example Code

// Primitive Types
let name = "John";     // String
let age = 25;          // Number
let isActive = true;   // Boolean
let x;                 // Undefined
let y = null;          // Null

// Non-Primitive
let user = {name: "John"};  // Object
let numbers = [1,2,3];      // Array

function greet() {
    return "Hello!";
}

// Special Types
let bigNum = 123456789n;    // BigInt
let id = Symbol("id");      // Symbol

📘 Type Checking (typeof)

typeof "Hello"      // string
typeof 10           // number
typeof true         // boolean
typeof undefined    // undefined
typeof null         // object (JS bug)
typeof {}           // object
typeof []           // object
typeof function(){} // function

📘 var vs let vs const

Keyword Scope Redeclare Reassign
var Function Scope Yes Yes
let Block Scope No Yes
const Block Scope No No 
// var
var a = 10;
var a = 20; // allowed

// let
let b = 10;
b = 30; // allowed

// const
const c = 10;
// c = 20; ❌ not allowed

📘 Hoisting

Definition: JavaScript moves variable declarations to the top.

// var (hoisted)
console.log(a); // undefined
var a = 10;

// let (TDZ - Error)
console.log(b); // ❌ error
let b = 10;

🎯 Interview Questions

1. What are Data Types?

They define what kind of value a variable can store.

2. null vs undefined?

  • undefined → no value assigned
  • null → intentionally empty

3. Why typeof null = object?

This is a JavaScript bug.

4. var vs let vs const?

var → function scope, let & const → block scope

5. What is hoisting?

Variables are moved to top during execution.

6. Can const object change?

Yes, properties can change.

const obj = {name: "A"};
obj.name = "B"; // allowed
