📘 JavaScript Data Types

🔹 Types of Data Types

  • Primitive Data Types
    • String
    • Number
    • Boolean
    • Undefined
    • Null
    • BigInt
    • Symbol
  • Non-Primitive Data Types
    • Object
    • Array
    • Function

🔹 Explanation

  • String: Text inside quotes ("Hello")
  • Number: Integer & decimal (10, 10.5)
  • Boolean: true / false
  • Undefined: Variable declared but not assigned
  • Null: Intentional empty value
  • Object: Key-value pair collection
  • Array: Ordered list
  • Function: Reusable block of code
  • BigInt: Large numbers
  • Symbol: Unique identifier

🔹 Examples

let str = "Hello";        // String
let num = 100;            // Number
let bool = true;          // Boolean
let x;                    // Undefined
let y = null;             // Null

let obj = {name:"John"};  // Object
let arr = [1,2,3];        // Array

function test() {         // Function
    return "Hi";
}

let big = 123456789n;     // BigInt
let sym = Symbol("id");   // Symbol

📘 Type Checking

typeof "Hello"   // string
typeof 10        // number
typeof true      // boolean
typeof undefined // undefined
typeof null      // object (JS bug)
typeof {}        // object
typeof []        // object
typeof function(){} // function

📘 var vs let vs const

Keyword Scope Redeclare Reassign
var Function Yes Yes
let Block No Yes
const Block No No

🔹 Examples

// var
var a = 10;
var a = 20; // allowed

// let
let b = 10;
// let b = 20; ❌ error
b = 30;

// const
const c = 10;
// c = 20; ❌ error

📘 Hoisting

Hoisting means variables are moved to the top during execution.

console.log(a);
var a = 10;   // undefined (hoisted)

console.log(b);
let b = 10;   // ❌ error (TDZ)

🎯 Interview Questions & Answers

1. What are data types in JavaScript?

Data types define the type of data a variable can hold (string, number, boolean, etc.).

2. Difference between null and undefined?

  • undefined → variable declared but not assigned
  • null → intentionally empty value

3. Why typeof null is object?

This is a known bug in JavaScript.

4. Difference between var, let, const?

var → function scope, let & const → block scope

5. What is hoisting?

Moving declarations to the top of scope.

6. Can const object be modified?

Yes, properties can change but reference cannot.

const obj = {name:"A"};
obj.name = "B"; // ✅ allowed

7. What is block scope?

Scope limited to {} block.

8. Difference between == and ===?

  • == → checks value
  • === → checks value + type

9. Is array an object?

Yes, array is a special type of object.

10. What is BigInt?

Used for very large numbers beyond Number limit.
